# Contributing to Retreever

Thank you for your interest in contributing to **Retreever**! The project is intentionally lightweight and developer-focused, and contributions that keep it simple, clear, and useful are always welcome.

## Getting Started

1. Fork the repository.
2. Clone your fork and create a new branch for your change.
3. Make your updates with clear, meaningful commit messages.
4. Add tests if your change affects behavior.
5. Submit a pull request to the `main` branch, including a descriptive explanation of your changes.

## Code Style & Guidelines

- Follow the existing structure and conventions in the project.
- Keep implementations small, focused, and readable.
- Public classes and methods should include brief Javadoc.
- Avoid unnecessary complexity—Retreever prioritizes simplicity.
- Use explicit imports (avoid `*`).
- Include license headers on new source files.

## Reporting Issues

Use GitHub Issues for bugs, feature requests, or questions. Please include enough detail to reproduce the problem (Spring version, stack trace, steps, etc.).

## Development Setup

- Java **17+**
- Maven

Build the project:

``` bash
  mvn clean package
```

Run tests:

``` bash
  mvn test
```


## Review Process

Pull requests are reviewed for clarity, correctness, and consistency. You might be asked to refine code or add missing details—this is a normal part of the process.
